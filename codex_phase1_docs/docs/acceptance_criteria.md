# acceptance_criteria.md

## Phase1 完了条件

以下をすべて満たしたら Phase1 完了とみなす。

### 1. 中間入力
- 1商品1行の中間CSVまたは中間シートで運用できる
- 最低限必要な入力列だけで成り立つ
- AI提案値と人の上書き値を両方扱える

### 2. 確認工程
- GAS が確認用シートまたは確認用CSVを生成できる
- 人が最終確認し、必要箇所だけ修正できる
- `publish_flag` で出力対象/非対象を切り替えられる

### 3. バリデーション
- 商品コード不正を検出できる
- 必須項目不足を検出できる
- 価格不正を検出できる
- カテゴリ形式不正を検出できる
- 画像生成条件の不整合を検出できる
- エラー行を出力対象から外せる

### 4. 画像
- 画像URLを自動生成できる
- `product_code` と画像フォルダを一致前提で扱える
- `image_count` に応じて画像1〜20を埋められる
- 白背景画像の有無を制御できる

### 5. ir-item.csv 出力
- `ir-item.csv` を単独で生成できる
- A/B/C列を必ず出力できる
- 公式テンプレートのヘッダ順を維持できる
- Shift_JIS / CRLF で出力できる
- item Robot にインポート可能な形式で出力できる

### 6. 実装範囲の制御
- itemsub のコードが混ざっていない
- バリエーションCSVのコードが混ざっていない
- 再ダウンロード前提のロジックが標準フローに入っていない

### 7. ドキュメント
- README がある
- 中間CSV → `ir-item.csv` のマッピング表がある
- 固定値一覧がある
- 将来拡張ポイントが明記されている

## 非完了条件
以下のどれかが残る場合は未完了。

- ヘッダ順が不明なまま実装している
- 商品コードルールが曖昧
- 人の確認ステップが無い
- 画像URLが手入力前提
- itemsub やバリエーションのロジックが混入している
