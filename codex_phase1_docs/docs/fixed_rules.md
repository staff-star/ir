# fixed_rules.md

## 1. 商品コード
- item Robot の商品コード（楽天URL）をメインキーとする
- 32byte以内
- 使用可能文字は `0-9`, `a-z`, `-`, `_`
- 全角・空白・大文字は許可しない
- 一意であること

## 2. A/B/C列
`ir-item.csv` の A/B/C列は必須。

- A列: 商品コード（楽天URL）
- B列: メインデータの商品名
- C列: ショップ名

### 運用ルール
- B列は値が空でも列は削らない
- C列は固定値で出力する

## 3. カテゴリ形式
カテゴリは item Robot カテゴリID をパイプ形式で出力する。

例:
- `|683|`
- `|683|530|`

ルール:
- 指定するカテゴリIDが1つでもパイプで囲む
- 複数ある場合は横に連結する
- 最下層カテゴリIDを使う

## 4. 商品名
- `ir-item.csv` の商品名は 127文字以内
- 優先順位:
  1. `final_title_override`
  2. `ai_title`
  3. `raw_name`

## 5. 価格
- 販売価格は半角数字 9文字以内
- 表示価格は Phase1 では販売価格と同値を基本とする
- 必要なら将来 `-1` / `-2` 運用へ拡張可能にする

## 6. 税・送料・在庫
Phase1 では固定値または運用固定値で管理する。

### 想定固定値候補
- 消費税: `1`
- 送料: 運用固定
- 在庫タイプ: 単品前提では `1`
- 新品・中古: `0`

### 注意
- 税率と軽減税率は `tax_class` から分岐できるようにする
- 食品/非食品混在の余地を残す

## 7. 画像
- 画像URLは item Robot 形式URLを使う
- GASで自動生成する
- `image_count` と生成画像数が一致すること
- `has_white_image = 1` のときのみ `w.{ext}` を生成する

## 8. 出力対象
- `publish_flag = 1` の行のみ出力する
- エラーがある行は出力しない
- エラー一覧へ分離する

## 9. 出力形式
- 文字コード: Shift_JIS
- 改行コード: CRLF
- 非圧縮CSV
- 公式テンプレートのヘッダ順を崩さない

## 10. ヘッダ正本
正本は item Robot アカウントからダウンロードした最新の `ir-item.csv` テンプレートとする。
実装時は手元の参考ヘッダと比較し、差分があれば公式テンプレートを優先する。
